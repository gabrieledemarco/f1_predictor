F1 PREDICTOR - ISTRUZIONI DI ESECUZIONE
========================================

## ANALISI DEL PROGETTO

**Struttura del progetto:**
```
f1_predictor/
├── f1_predictor/           # Package principale
│   ├── models/            # 4 layer ML (TrueSkill, Kalman, Bayesian, Ensemble)
│   ├── calibration/       # Calibrazione quote (devig, isotonic, edge tracking)
│   ├── validation/        # Walk‑forward, backtesting, metriche
│   ├── data/              # Loader per Jolpica, TracingInsights, quote Pinnacle
│   ├── domain/            # Entità di dominio (Race, Driver, etc.)
│   ├── pipeline/          # Orchestratore principale (F1PredictionPipeline)
│   ├── reports/           # Generazione report edge
│   └── train_pipeline.py  # Script di training CLI
├── pipeline.py            # Copia della pipeline (duplicato)
├── F1_Predictor_Documentazione_Tecnica.docx
└── data/racedata/         # Dataset storico (CSV da TracingInsights)
```

**Scopo:** Sistema di predizione multistrato per Formula 1 che combina:
1. **TrueSkill Through Time** – rating piloti condizionato al tipo di circuito
2. **Filtro di Kalman** – stima del passo macchina per costruttore
3. **Simulazione Bayesiana** – Monte Carlo per probabilità di vittoria/podio/DNF
4. **Ensemble Ridge** – correzione meta‑learner
5. **Calibrazione isotonica** – allineamento alle quote Pinnacle
6. **Calcolo edge** – identificazione valore scommesse

## CONFIGURAZIONE MONGODB (OPZIONALE)

Il progetto supporta il salvataggio degli artefatti di training su MongoDB via GridFS.
Per abilitare questa funzionalità:

1. Crea un file `.env` nella directory principale del progetto:
```

```

2. Assicurati che il database sia accessibile.

**Nota:** Se non configuri MongoDB, usa il flag `--dry-run` durante il training.

## ESECUZIONE DEL TRAINING

Lo script principale per il training è `f1_predictor/train_pipeline.py`.

### Comando base:
```bash
python f1_predictor/train_pipeline.py --year 2026 --through-round 5 --train-from 2019
```

### Parametri principali:
- `--year` - Anno GP corrente (es. 2026)
- `--through-round` - Ultimo round incluso nel training (es. 5)
- `--train-from` - Primo anno del dataset di training (default: 2019)
- `--val-from` - Primo anno del set di validazione walk‑forward (default: 2022)
- `--n-mc-sim` - Numero simulazioni Monte Carlo (default: 50000)
- `--ridge-alpha` - Regolarizzazione Ridge (default: 10.0)
- `--min-calib-obs` - Osservazioni minime per calibratore isotonic (default: 100)

### Flag utili:
- `--dry-run` - Esegue training ma non salva su MongoDB
- `--synthetic` - Forza uso dati sintetici (test/sviluppo)
- `--list-versions` - Lista versioni modello su MongoDB ed esce
- `--rollback v20260303_0900` - Rollback a versione specifica
- `--delete-old 5` - Elimina versioni vecchie, mantieni ultime 5

### Esempi di training:
```bash
# Training con dati reali
python f1_predictor/train_pipeline.py --year 2026 --through-round 5 --train-from 2019

# Training di prova (senza MongoDB)
python f1_predictor/train_pipeline.py --year 2026 --through-round 3 --synthetic --dry-run

# Lista versioni disponibili
python f1_predictor/train_pipeline.py --list-versions

# Rollback a versione specifica
python f1_predictor/train_pipeline.py --rollback v20260303_0900
```

## UTILIZZO DELLA PIPELINE PER PREDIZIONI

### Esempio di script Python (`predict_example.py`):

```python
from f1_predictor.pipeline import F1PredictionPipeline
from f1_predictor.domain.entities import Race, Circuit, CircuitType

# 1. Carica dati storici (esempio con dati sintetici)
from f1_predictor.data.adapter import generate_seasons
races = generate_seasons(years=[2024, 2025], through_round=5)

# 2. Addestra pipeline
pipeline = F1PredictionPipeline()
pipeline.fit(races, verbose=True)

# 3. Prepara input per una gara futura
race = Race(
    race_id=2026006,
    name="GP Australia",
    circuit=Circuit(
        circuit_ref="albert_park",
        circuit_type=CircuitType.STREET,
        country="Australia"
    ),
    year=2026,
    round_num=6,
    date="2026-03-23"
)

driver_grid = [
    {"driver_code": "VER", "constructor_ref": "red_bull", "grid_position": 1},
    {"driver_code": "LEC", "constructor_ref": "ferrari", "grid_position": 2},
    # ... aggiungi altri piloti
]

pinnacle_odds = {"VER": 1.85, "LEC": 3.50, "NOR": 5.00, "RUS": 8.00}

# 4. Esegui predizione
result = pipeline.predict_race(race, driver_grid, pinnacle_odds, verbose=True)

# 5. Analizza risultati
print("=== PROBABILITÀ CALCOLATE ===")
for driver_code, prob in result["probabilities"].items():
    print(f"{driver_code}: P(win)={prob.p_win:.3f}, P(podium)={prob.p_podium:.3f}")

if result["edge_report"]:
    print("\n=== EDGE REPORT ===")
    print(f"Selezioni con edge > 4%: {len(result['edge_report']['positive_edge'])}")
    for sel in result["edge_report"]["positive_edge"]:
        print(f"  {sel['driver']}: quota={sel['odds']:.2f}, edge={sel['edge']:.2%}")
```

## VALIDAZIONE E BACKTESTING

### Walk‑forward validation:
```python
from f1_predictor.validation.walk_forward import WalkForwardValidator

validator = WalkForwardValidator()
metrics = validator.evaluate(
    races,                    # lista di gare storiche
    train_from=2020,          # primo anno training
    val_from=2023,            # primo anno validazione
    n_mc_sim=50000,
    ridge_alpha=10.0
)

print(f"Brier Score: {metrics['brier']:.4f}")
print(f"Kendall tau: {metrics['kendall_tau']:.3f}")
print(f"ROI simulato: {metrics['roi']:.1f}%")
print(f"ECE: {metrics['ece']:.4f}")
```

### Backtesting:
```python
from f1_predictor.validation.backtesting import BettingBacktester, BacktestConfig

config = BacktestConfig(
    kelly_fraction=0.25,
    edge_threshold=0.04,
    bankroll=1000.0
)
backtester = BettingBacktester(config=config)

# Simula scommesse storiche
backtest_results = backtester.run(
    historical_predictions,   # predizioni passate
    historical_odds,          # quote storiche
    historical_outcomes       # risultati reali
)

print(f"ROI finale: {backtest_results['final_roi']:.1f}%")
print(f"Sharpe ratio: {backtest_results['sharpe_ratio']:.2f}")
```

## RISOLUZIONE PROBLEMI

### 1. `ModuleNotFoundError: No module named 'core.db'`
**Problema:** Il modulo `core.db` non è presente nel progetto.
**Soluzione:** Usa `--dry-run` per evitare la scrittura su MongoDB, oppure crea un file `core/db.py` con una connessione fittizia.

### 2. Dati storici insufficienti
**Problema:** I dataset reali non sono disponibili o incompleti.
**Soluzione:** Il loader passa automaticamente a dati sintetici. Puoi forzare l'uso di dati sintetici con `--synthetic`.

### 3. MongoDB non configurato
**Problema:** Errore di connessione a MongoDB.
**Soluzione:** 
- Usa `--dry-run` per training senza salvataggio
- Configura correttamente `MONGO_URI` nel file `.env`
- Verifica che il cluster MongoDB sia accessibile

### 4. Quote Pinnacle mancanti
**Problema:** La calibrazione isotonica (Layer 4) richiede almeno 100 osservazioni.
**Soluzione:** 
- Aggiungi dati di quote reali in `data/pinnacle_odds/` (formato JSONL)
- Usa il modello senza calibrazione (funziona comunque)
- Il calibratore si attiverà automaticamente quando avrà abbastanza dati

### 5. Training lento
**Problema:** Le simulazioni Monte Carlo (50k+ default) possono essere pesanti.
**Soluzione:** 
- Riduci `--n-mc-sim` a 10000-20000 per testing
- Usa `--synthetic` per dati più leggeri
- Durante walk‑forward validation, il numero di simulazioni viene automaticamente ridotto

## METRICHE DI PERFORMANCE ATTESE

Dal walk‑forward validation, obiettivi ragionevoli:
- **Brier Score**: < 0.20 (minore è meglio)
- **Kendall tau**: > 0.45 (correlazione ranking predetto/reale)
- **ROI simulato**: > 0% (profitto atteso)
- **ECE (Expected Calibration Error)**: < 0.05 (calibrazione accurata)
